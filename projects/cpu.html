Processor Design

A 32-bit Two-Cycle Processor

The processor has a 2-stage pipeline:

  • Instruction Fetch: Instruction is fetched from instruction memory.
  • Execute: The instruction is decoded, executed, and committed (written back). This is a combination of the remaining stages of a normal MIPS pipeline.

    • With that two-cycle pipline, the processor implements the Instruction Set Architecture (ISA) using a two-cycle pipeline. It contains an instance Arithmetic Logic Unit (ALU), Data Memory, and Register File with the entire datapath and control.

    The processor gets its program from the processor harness we have provided in run.circ. It send the address of instruction memory it wants to access to the harness through an output, and accept the instruction at that address as an input.
