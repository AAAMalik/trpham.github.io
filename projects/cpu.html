Processor Design

A 32-bit two-cycle processor

Your processor will have a 2-stage pipeline: Instruction Fetch: An instruction is fetched from the instruction memory. Execute: The instruction is decoded, executed, and committed (written back). This is a combination of the remaining stages of a normal MIPS pipeline. Your completed processor should implement the Instruction Set Architecture (ISA) using a two-cycle pipeline. Your processor will contain an instance of yourRegister Files, ALUArithmetic Logic Unit (ALU) [hide], Data Memory, and Register File. You are also responsible for constructing the entire datapath and control from scratch. Your processor will get its program from the processor harness we have provided in run.circ. It will send the address of instruction memory it wants to access to the harness through an output, and accept the instruction at that address as an input.
